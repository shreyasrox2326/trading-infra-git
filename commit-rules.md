# Commit / Branch Rules

| Commit type | Branch prefix | Meaning |
|---|---|---|
| `docs` | `docs/` | documentation / notes |
| `data` | `data/` | market data schema or pipeline |
| `strategy` | `strategy/` | strategy configs or logic |
| `backtest` | `backtest/` | backtest runner / replay / evaluation |
| `paper` | `paper/` | daily paper-trading workflow |
| `infra` | `infra/` | GitHub Actions, R2, deployment |
| `model` | `model/` | training, inference, model handling |
| `fix` | `fix/` | bug fixes |
| `chore` | `chore/` | cleanup, formatting, dependencies |
